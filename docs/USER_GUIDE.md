## Установка и запуск

### Требования

- Windows 10/11
- Python 3.11 (для запуска через launcher)

### Запуск

1. Распакуйте архив с приложением.
2. Запустите файл `start.bat`.

Приложение автоматически запустит backend и откроет браузер.

## Импорт данных

### Формат файлов

Приложение принимает текстовые файлы с расширением `.dat`.

Формат строки:

```text
DD.MM.YYYY HH:MM:SS.FFF значение
```

Пример:

```text
01.01.2020 00:00:00.000 -1234.567
01.01.2020 00:01:00.000 -1235.123
01.01.2020 00:02:00.000 -1234.890
```

Требования:

- разделитель — пробел
- дата и время — `DD.MM.YYYY HH:MM:SS.FFF` или `DD.MM.YYYY HH:MM:SS`
- значение — число с плавающей точкой
- кодировка — UTF-8

### Процедура импорта

1. Нажмите кнопку **Импорт**.
2. В диалоге выберите один или несколько `.dat` файлов.
3. При необходимости включите фильтр выбросов (см. ниже).
4. Дождитесь завершения импорта.

После импорта приложение автоматически обновит диапазон дат и пересчитает график.

### Повторный импорт

Если файл уже импортирован, приложение пропустит его.
Чтобы обновить данные, удалите старые записи или импортируйте файл с другим именем.

## Работа с графиком

### Элементы управления

Над графиком расположены кнопки:

| Кнопка | Действие |
|--------|----------|
| **←** | Сдвинуть влево |
| **−** | Уменьшить масштаб |
| **+** | Увеличить масштаб |
| **→** | Сдвинуть вправо |
| **Все** | Показать весь период |

### Масштабирование

- **Выделение мышью** — зажмите левую кнопку мыши и выделите область для приближения
- **Колёсико мыши** — прокрутка для масштабирования по горизонтали
- **Кнопки +/−** — пошаговое изменение масштаба

### Подсказки

Наведите курсор на точку графика, чтобы увидеть:

- дату и время
- среднее значение
- стандартное отклонение
- минимум и максимум
- количество записей в точке

### Разрывы на графике

Если в данных есть пропуски (нет измерений за период), на графике отображается разрыв — линия прерывается.

## Выбор периода и дискретности

### Дискретность (частота агрегации)

| Значение | Описание | Пример использования |
|----------|----------|----------------------|
| **Секунда** | По каждой секунде | Детальный анализ |
| **10 минут** | 10-минутные интервалы | Краткосрочные изменения |
| **Час** | По часам | Суточная динамика |
| **День** | По дням | Основной режим |
| **Неделя** | По неделям | Среднесрочные тренды |
| **Месяц** | По месяцам | Сезонные изменения |
| **Квартал** | По кварталам | Квартальная отчётность |
| **Год** | По годам | Долгосрочные тренды |
| **Декада** | 1–10, 11–20, 21–конец месяца | Месячный анализ |

### Быстрый выбор периода

Кнопки над полями дат:

- **Весь** — весь доступный период
- **30д** — последние 30 дней
- **7д** — последние 7 дней
- **1д** — последние сутки

### Ручной ввод

1. Установите даты в полях **С** и **По**.
2. Выберите **Дискретность**.
3. Нажмите **Рассчитать**.

> **Совет:** Для больших периодов (несколько лет) выбирайте дискретность «День» или крупнее. При «Секунда» или «Час» расчёт может занять время.

## Фильтрация выбросов

При импорте можно включить автоматическую очистку данных от ошибочных измерений.

### Как включить

1. Перед импортом нажмите кнопку **Фильтр** — она станет активной.
2. Нажмите **Импорт** и выберите файлы.

### Принцип работы

Фильтр работает в два этапа:

1. **Глобальный IQR** — удаляет экстремальные значения, сильно выпадающие за пределы общего диапазона
2. **Локальный Hampel** — удаляет точечные выбросы (пики и провалы) на основе соседних значений

### Результат

После импорта в статусе отобразится:

- количество обработанных файлов
- количество импортированных записей
- количество удалённых выбросов

> **Внимание:** Фильтр применяется только при импорте. Уже загруженные данные не изменяются.

## Экспорт данных

### Экспорт агрегированных данных

1. Постройте график с нужным периодом и дискретностью.
2. Нажмите **Экспорт**.
3. Файл CSV автоматически сохранится в папку загрузок браузера.

Формат CSV:

```csv
datetime,mean,std,min,max,count
2020-01-01T00:00:00,-1234.56,12.34,-1250.00,-1220.00,1440
```

### Экспорт месяца

1. Нажмите **Экспорт месяца**.
2. Выберите год и месяц.
3. Нажмите **Экспорт**.

Результат — файл `YYYY_MM.dat` с сырыми данными в исходном формате.

## Советы и рекомендации

### Производительность

| Ситуация | Рекомендация |
|----------|--------------|
| Импорт >10 файлов | Используйте фильтр, данные очистятся автоматически |
| Период >5 лет | Выбирайте дискретность «День» или крупнее |
| График тормозит | Увеличьте дискретность — меньше точек = быстрее |
| Нужны детали | Уменьшите период до 7–30 дней и выберите «Час» |

### Формат дат

При ручном вводе дат используйте формат `ГГГГ-ММ-ДД` (например, `2020-01-15`).

### Сохранение базы данных

Файл `sealevel.duckdb` содержит все импортированные данные.
Скопируйте его для резервного сохранения.

### Частые проблемы

**График пустой**

- Проверьте, что данные импортированы (кнопка **Файлы**)
- Убедитесь, что выбран корректный период
- Нажмите **Весь**, чтобы показать весь диапазон

**Импорт не работает**

- Проверьте формат файла (см. раздел [Импорт данных](#импорт-данных))
- Убедитесь, что файл имеет расширение `.dat`
- Проверьте кодировку файла (должна быть UTF-8)

**Долгий расчёт**

- Увеличьте дискретность
- Уменьшите период
- Для секундной дискретности ограничьте период 1–2 днями
