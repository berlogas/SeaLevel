# Фильтрация данных при импорте

## Что такое квартили

**Квартили** — это значения, которые делят упорядоченный набор данных на четыре равные части.

Представьте, что вы выстроили все измерения по порядку (от самого маленького к самому большому):

```
Q1              Q2 (медиана)              Q3
 |                  |                      |
 v                  v                      v
[====|====|====|====|====|====|====|====|====]
0%   25%  50%       75%                  100%
```

| Квартиль | Название | Описание |
|----------|----------|----------|
| **Q1** | Первый квартиль | Значение, ниже которого находится 25% данных |
| **Q2** | Медиана | Значение, ниже которого находится 50% данных |
| **Q3** | Третий квартиль | Значение, ниже которого находится 75% данных |

### Пример

```
Измерения уровня моря (в мм): -1200, -1150, -1100, -1050, -1000, -950, -900

Q1 (25-й перцентиль) = -1100
Q2 (медиана)         = -1050
Q3 (75-й перцентиль) = -1000
```

**IQR** (Interquartile Range) — это расстояние между Q1 и Q3:
```
IQR = Q3 - Q1 = -1000 - (-1100) = 100
```

### Зачем это нужно для фильтрации

Квартили описывают "типичный" разброс данных. Значения, которые сильно выходят за пределы этого разброса, скорее всего являются выбросами или ошибками измерений.

### Визуально

```
Значение:  -1200  -1100  -1050  -1000  -950   -900   -500
             |       |      |      |      |      |      |
Доля:       0%    25%    50%    75%   100%
                    Q1   Q2(med)   Q3

IQR = Q3 - Q1 = -950 - (-1100) = 150

Границы для фильтра:
  Q1 - 3×IQR = -1100 - 450 = -1550
  Q3 + 3×IQR = -950 + 450 = -500

  Значение -500 выходит за верхнюю границу -> ВЫБРОС
```

## Общая схема

При импорте файла с включённым фильтром (🔘) данные проходят **двухуровневую очистку**:

```
┌─────────────────────────────────────────────────────────────┐
│                    Исходные данные                          │
│                    (например, 46 217 737 записей)            │
└──────────────────────┬──────────────────────────────────────┘
                       │
                       ▼
┌─────────────────────────────────────────────────────────────┐
│              Уровень 1: Глобальный IQR фильтр                 │
│                                                             │
│  • Вычисляет квартили Q1, Q3 по ВСЕМ данным                 │
│  • IQR = Q3 - Q1                                            │
│  • Границы: [Q1 - 3×IQR , Q3 + 3×IQR]                        │
│  • Удаляет ВСЕ точки за пределами этих границ               │
└──────────────────────┬──────────────────────────────────────┘
                       │
                       ▼
┌─────────────────────────────────────────────────────────────┐
│              Уровень 2: Локальный Hampel фильтр             │
│                                                             │
│  • Для каждой точки вычисляет окно ±50 соседних значений    │
│  • MEDIAN(window) — медиана окна                             │
│  • MAD(window) — Median Absolute Deviation                   │
│  • Удаляет если: |value - median| > k × MAD                 │
└──────────────────────┬──────────────────────────────────────┘
                       │
                       ▼
┌─────────────────────────────────────────────────────────────┐
│                    Очищенные данные                          │
│                    (например, 43 603 382 записи)             │
└─────────────────────────────────────────────────────────────┘
```

## Уровень 1: Глобальный IQR фильтр

### Принцип работы

IQR (Interquartile Range) — мера статистического разброса данных.

```
         Q1-3×IQR      Q1     Q3      Q3+3×IQR
            |          |      |          |
    ────────┼──────────┼──────┼──────────┼────────
    ════════════════════════════════════════════════
            |<──── IQR ────>|        выбросы
```

### Формула

```
IQR = Q3 - Q1
Нижняя граница = Q1 - 3 × IQR
Верхняя граница = Q3 + 3 × IQR
```

### Особенности

- **Глобальный** — анализирует все данные сразу
- **Устойчив к выбросам** — сами выбросы не влияют на квартили
- **Удаляет экстремальные значения** — сдвиги, ступеньки, артефакты

### Пример

```
Данные: min=-2450, max=-300, Q1=-1350, Q3=-1150
IQR = -1150 - (-1350) = 200
Нижняя граница = -1350 - 3×200 = -1950
Верхняя граница = -1150 + 3×200 = -550

Результат: удалены все значения > -550 и < -1950
```

## Уровень 2: Локальный Hampel фильтр

### Принцип работы

Hampel filter — робастный метод обнаружения выбросов на основе медианы.

```
    Окно из 101 точки (±50)
    ┌────────────────────────┐
    │                        │
    ▼                        ▼
    [5] [3] [7] [2] [8] [500] [4] [6] [9] ...
                    ▲
                  выброс
```

### Формула

```
MAD = median(|xi - median(window)|)
Выброс если: |value - median(window)| > k × MAD
```

### Параметры

| Параметр | Значение | Описание |
|----------|----------|----------|
| `half_window` | 50 | Размер окна с каждой стороны (±50 = 101 точка) |
| `k` | 3.0 | Множитель для MAD (порог чувствительности) |

### Особенности

- **Локальный** — анализирует каждую точку в контексте соседей
- **Робастный** — медиана нечувствительна к выбросам в окне
- **Удаляет точечные аномалии** — пики, провалы, шумовые выбросы

### Пример

```
Окно: [-1200, -1190, -1185, -1210, -1175, -500, -1200, ...]
median = -1200
Отклонения: [0, 10, 15, 10, 25, 700, 0, ...]
MAD = median([0, 10, 15, 10, 25, 700, 0, ...]) = 15
Порог = 3 × 15 = 45

Значение -500:
|-500 - (-1200)| = 700 > 45 → ВЫБРОС
```

## Когда какой фильтр применять

### Только IQR фильтр
- Данные содержат **глобальные сдвиги** (ступеньки, тренды)
- Много выбросов распределено по всей амплитуде
- Данные сильно загрязнены артефактами

### Только Hampel фильтр
- Выбросы **локальные** (отдельные пики и провалы)
- Данные в целом чистые
- Нужно сгладить шум

### Оба фильтра (по умолчанию)
- Данные содержат **смесь** глобальных и локальных выбросов
- Сначала IQR удаляет экстремумы
- Затем Hampel очищает локальные аномалии

## Технические детали

### SQL запрос IQR фильтра

```sql
-- Вычисляем квартили
SELECT MIN(level), MAX(level), AVG(level),
       QUANTILE_CONT(level, 0.25),
       QUANTILE_CONT(level, 0.75)
FROM import_temp;

-- Удаляем выбросы
DELETE FROM import_temp
WHERE level < ? OR level > ?;
```

### SQL запрос Hampel фильтра

```sql
INSERT INTO sea_readings (timestamp_ms, level, source_file)
WITH windowed AS (
    SELECT 
        timestamp_ms,
        level,
        MEDIAN(level) OVER (
            ORDER BY timestamp_ms 
            ROWS BETWEEN 50 PRECEDING AND 50 FOLLOWING
        ) as window_median,
        MAD(level) OVER (
            ORDER BY timestamp_ms 
            ROWS BETWEEN 50 PRECEDING AND 50 FOLLOWING
        ) as window_mad
    FROM import_temp
)
SELECT timestamp_ms, level, 'filename'
FROM windowed
WHERE window_mad = 0 OR ABS(level - window_median) <= 3.0 * window_mad;
```

## Регулировка чувствительности

### Увеличение k (по умолчанию 3.0)
```
k = 4.0 — мягче, меньше удалений
k = 2.0 — жёстче, больше удалений
k = 1.5 — агрессивно, только явные выбросы
```

### Уменьшение IQR множителя
```
iqr_mult = 2.0 — шире границы, меньше глобальных удалений
iqr_mult = 4.0 — уже границы, больше глобальных удалений
```

## Заключение

Двухуровневая фильтрация обеспечивает **комплексную очистку данных**:
1. **IQR** — удаляет глобальные аномалии и экстремумы
2. **Hampel** — очищает локальные выбросы и шум

Результат: чистый временной ряд без артефактов.
